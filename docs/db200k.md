# DB200K Scanning

This repo ships a small, self-contained DB200K toolchain for:
- raw sequence-window scans against DB200K-derived query profiles
- sequence-aware reranking with simple exposure/flexibility heuristics
- exact rank checks for fixed windows
- chunked scans across large FASTA collections

DB200K fragment PDBs and energy tables are not bundled. Download `DB200K.tar.gz`
from the supplemental files of the Protein Science paper, extract it, and pass
the extracted directory as `--db-root`.

## What You Need

Every scan needs:
- a query sequence
- an extracted DB200K root passed with `--db-root`
- a FASTA to scan

The repo-local example FASTA is:
- [examples/db200k/nbr1_human_uba_913_966.fasta](../examples/db200k/nbr1_human_uba_913_966.fasta)

## Example: MVP Loop Against the NBR1 UBA Domain

Core loop query:
- `MVP 607-623`
- `ETSEAKGPDGMALPRPR`

Extended loop query:
- `MVP 604-626`
- `FGFETSEAKGPDGMALPRPRDQA`

Target:
- `NBR1 UBA 913-966`

### 1. Raw Scan of the Core Loop

```bash
python scripts/scan_db200k.py \
  --query-seq ETSEAKGPDGMALPRPR \
  --db-root /path/to/pisces-cache-scaac \
  --fasta examples/db200k/nbr1_human_uba_913_966.fasta \
  --top-k 10
```

This should recover the expected UBA-centered window family around:
- `LRLLKKHNYNILQVVTE`

### 2. Extend the Same Register

```bash
python scripts/scan_db200k.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta examples/db200k/nbr1_human_uba_913_966.fasta \
  --alignment-mode trim_query_one_target_gap \
  --target-flank 1 \
  --peripheral-flank-weight 0.5 \
  --top-k 10
```

With the current scoring rules, the top hit expands into the same UBA
neighborhood, centered on:
- `LNLRLLKKHNYNILQVVTELLQL`

### 3. Sequence-Aware Rerank

Use this when you want loop-like, exposed-looking windows to outrank buried,
hydrophobic ones without requiring a structure.

```bash
python scripts/scan_db200k_accessibility.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta examples/db200k/nbr1_human_uba_913_966.fasta \
  --alignment-mode trim_query_one_target_gap \
  --target-flank 1 \
  --peripheral-flank-weight 0.5 \
  --top-k 5
```

The key combined score is:

```text
sequence_rank_score = db200k_score - seq_bonus
```

More negative is better.

### 4. Proteome-Scale Scan With Heuristic Prefiltering

For large FASTAs, run a cheap heuristic screen first and only DB200K-score the
best sequence-side candidates.

```bash
python scripts/scan_db200k_accessibility.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta swissprot.fasta \
  --heuristic-prefilter-top-n 2000 \
  --alignment-mode rigid \
  --top-k 200
```

Useful first pass:

```bash
python scripts/scan_db200k_accessibility.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta swissprot.fasta \
  --report-heuristic-only
```

### 5. Exact Rank of Fixed Windows

When the question is "where does this exact target window rank?", use
`scripts/report_db200k_window_ranks.py`.

```bash
python scripts/report_db200k_window_ranks.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta swissprot.fasta \
  --alignment-mode rigid \
  --target uba_934_956=LNLRLLKKHNYNILQVVTELLQL
```

## Script Roles

### `scripts/scan_db200k.py`

Use this when:
- you want the raw DB200K result first
- you want to inspect alignment trimming or skipped residues directly
- you want per-position breakdowns via `--show-breakdown`

### `scripts/scan_db200k_accessibility.py`

Use this when:
- high raw DB200K hits need to be filtered by sequence-side accessibility or flexibility proxies
- you want heuristic prefiltering on very large FASTAs
- you want a final sequence-aware combined score without any structure dependency

### `scripts/report_db200k_window_ranks.py`

Use this when:
- you already have one or more target windows
- you need exact global rank rather than only the top returned hits

### `scripts/scan_db200k_chunks.py`

Use this when:
- your search set is already split into many FASTA chunks
- you want thresholded scans merged into a single sorted output file

### `scripts/rescore_db200k_reciprocal_3x3.py`

Use this when:
- you want a second-pass reciprocal 3x3 comparison on top DB200K hits

## Practical Defaults

Reasonable first-pass defaults:
- `--alignment-mode trim_query_one_target_gap`
- `--target-flank 1`
- `--peripheral-flank-weight 0.5`
- `--score-mode raw`

Use `rigid` first only when the register itself is the question.

## Interpreting Scores

The DB200K-oriented scripts in this repo use the same sign convention:
- more negative scores are better

That applies to:
- `db200k_score`
- `sequence_rank_score`
- reciprocal rescoring totals
