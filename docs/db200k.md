# DB200K Scanning

This repo includes a small DB200K toolchain for scanning FASTA windows against
DB200K-derived query profiles and then reranking hits with sequence or structure
features.

The most useful scripts are:
- `scripts/scan_db200k.py`
- `scripts/scan_db200k_accessibility.py`
- `scripts/report_db200k_window_ranks.py`
- `scripts/rerank_db200k_contact_weighted.py`

## What You Need

DB200K scans need:
- a query sequence
- an extracted DB200K root passed with `--db-root`
- a FASTA to scan

Optional structure-aware reranking also needs:
- a structure map TSV with columns `key,pdb_path,chain,seq_start,resseq_start`

The DB200K root is not versioned in this repo. In local work it has typically
been an extracted directory such as:

```bash
/path/to/pisces-cache-scaac
```

## Recommended Entry Points

Use `scripts/scan_db200k.py` when you want the raw DB200K scan and alignment
details. Default profile strategy is now `hierarchical_5x5_3x3_1x1` (no
per-position averaging).

Use `scripts/scan_db200k_accessibility.py` when you want to favor windows that
look more exposed and flexible, then optionally refine those hits with real RSA
and pLDDT from structures.

## Example: MVP Loop Against the NBR1 UBA Domain

The worked example below follows the biological direction that motivated this
analysis:

- first scan the MVP shoulder-loop core `607-623`
- then extend that same loop register and show that the hit expands within the
  NBR1 UBA-domain neighborhood

Core loop query:
- `MVP 607-623`
- `ETSEAKGPDGMALPRPR`

Extended loop query:
- `MVP 604-626`
- `FGFETSEAKGPDGMALPRPRDQA`

Target:
- `NBR1 UBA 913-959`
- repo-local FASTA contains only that domain segment for a fast reproducible
  example

Repo-local example inputs:
- FASTA: [examples/db200k/nbr1_human_uba_913_959.fasta](/Users/henryhardigan/Desktop/src/linear-complement-alignment/examples/db200k/nbr1_human_uba_913_959.fasta)
- structure map: [examples/db200k/nbr1_human_uba_structure_map.tsv](/Users/henryhardigan/Desktop/src/linear-complement-alignment/examples/db200k/nbr1_human_uba_structure_map.tsv)
- NBR1 AlphaFold model referenced by that map:
  [AF-Q14596-F1-model_v6.pdb](/Users/henryhardigan/Desktop/src/linear-complement-alignment/examples/db200k/AF-Q14596-F1-model_v6.pdb)

### 1. Core Loop Finds the UBA Motif Window

Start with the core loop only. This asks whether the `607-623` loop segment
lands in the expected UBA motif neighborhood at all.

```bash
python scripts/scan_db200k.py \
  --query-seq ETSEAKGPDGMALPRPR \
  --db-root /path/to/pisces-cache-scaac \
  --fasta examples/db200k/nbr1_human_uba_913_959.fasta \
  --alignment-mode trim_query_one_target_gap \
  --target-flank 1 \
  --peripheral-flank-weight 0.5 \
  --top-k 20
```

In the current local example data, the top hit is the expected motif-centered
UBA neighborhood:

- domain coordinates `24-40`
- full-protein coordinates `936-952`
- window `LRLLKKHNYNILQVVTE`

The same top-scoring local family also appears as adjacent overlapping windows
covering `NBR1 935-952` through `937-953`, which is what you want for a compact
motif-centered seed.

### 2. Extend the Same Alignment

Then extend the MVP query window while keeping the same target and alignment
mode. This shows whether the same UBA hit expands rather than disappearing.

```bash
python scripts/scan_db200k.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta examples/db200k/nbr1_human_uba_913_959.fasta \
  --alignment-mode trim_query_one_target_gap \
  --target-flank 1 \
  --peripheral-flank-weight 0.5 \
  --top-k 20
```

In the current local example data, the top hit expands cleanly into the longer
UBA window:

- domain coordinates `22-44` or `23-45`
- full-protein coordinates `934-956` or `935-957`
- top exact window in the domain FASTA:
  `LNLRLLKKHNYNILQVVTELLQL`

That is the same motif neighborhood as the core query, now recovered with the
longer flanking context.

### 3. Accessibility-Aware Rerank on the Extended Query

Once the extended alignment is established, add sequence and structure context.

```bash
python scripts/scan_db200k_accessibility.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta examples/db200k/nbr1_human_uba_913_959.fasta \
  --alignment-mode trim_query_one_target_gap \
  --target-flank 1 \
  --peripheral-flank-weight 0.5 \
  --structure-map examples/db200k/nbr1_human_uba_structure_map.tsv \
  --structure-top-k 5 \
  --top-k 5 \
  --out tmp/mvp_loop_vs_nbr1_uba_accessibility.tsv
```

In the repo-local example this keeps the same top UBA neighborhood and adds:
- `db200k_score`
- `seq_bonus`
- `sequence_rank_score`
- `rsa`
- `plddt`
- `struct_rank`

The key score relation is still:

```text
sequence_rank_score = db200k_score - seq_bonus
```

More negative is better.

### 4. Proteome-Scale Scan With Heuristic Prefiltering

For large FASTAs, scan time is dominated by DB200K scoring. The accessibility
script can prefilter by cheap sequence heuristics first, then run DB200K only on
the shortlisted windows.

```bash
python scripts/scan_db200k_accessibility.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta data/uniprot/sprot_human_from_chunks.fasta \
  --heuristic-prefilter-top-n 2000 \
  --alignment-mode rigid \
  --top-k 200 \
  --out tmp/mvp_loop_human_proteome_top2000.tsv
```

Useful rule of thumb:
- use `--report-heuristic-only` first if you want to measure how selective a
  threshold or `top-n` cutoff will be before paying for DB200K scoring

### 5. Exact Rank of Fixed NBR1 Windows

When the question is not "what are the top hits?" but "where does this exact
NBR1 UBA window rank against everything else?", use
`scripts/report_db200k_window_ranks.py`.

For the extended `23 aa` query:

```bash
python scripts/report_db200k_window_ranks.py \
  --query-seq FGFETSEAKGPDGMALPRPRDQA \
  --db-root /path/to/pisces-cache-scaac \
  --fasta data/uniprot/sprot_human_from_chunks.fasta \
  --alignment-mode rigid \
  --target uba_934_956=LNLRLLKKHNYNILQVVTELLQL
```

For the core `17 aa` query:

```bash
python scripts/report_db200k_window_ranks.py \
  --query-seq ETSEAKGPDGMALPRPR \
  --db-root /path/to/pisces-cache-scaac \
  --fasta data/uniprot/sprot_human_from_chunks.fasta \
  --alignment-mode rigid \
  --target uba_936_952=LRLLKKHNYNILQVVTE
```

This reports exact proteome-wide ranks under the same combined score used by the
accessibility scan.

## Script Roles

### `scripts/scan_db200k.py`

Use this when:
- you want the raw DB200K result first
- you want to inspect alignment trimming or skipped residues directly
- you do not want accessibility or structure reranking in the same step

### `scripts/scan_db200k_accessibility.py`

Use this when:
- high raw DB200K hits need to be filtered by accessibility or flexibility
- you want heuristic prefiltering on very large FASTAs
- you want optional structure-aware reranking from RSA and pLDDT

### `scripts/report_db200k_window_ranks.py`

Use this when:
- you already have one or more target windows
- you need exact global rank rather than only the top returned hits

### `scripts/rerank_db200k_contact_weighted.py`

Use this when:
- you want structure terms computed only on residues carrying favorable DB200K
  contributions
- you want contact-weighted RSA / disorder / protrusion reranking for a
  structure-backed target set

## Practical Defaults

Reasonable first-pass defaults:
- `--alignment-mode trim_query_one_target_gap`
- `--target-flank 1`
- `--peripheral-flank-weight 0.5`
- `--score-mode raw`

Use `rigid` first only when the register itself is the question.

## Interpreting Scores

The DB200K-oriented scripts in this repo use the same sign convention:
- more negative scores are better

That applies to:
- `db200k_score`
- `sequence_rank_score`
- structure-reranked composite scores

## Notes

- `python scripts/scan_db200k.py --help` and
  `python scripts/scan_db200k_accessibility.py --help` are still the most
  complete references for all flags.
- If cache building is slow the first time, point `--cache-dir` somewhere
  reusable or rebuild once with `--rebuild-cache`.
- These examples use repo-local NBR1 UBA files for convenience. Replace the
  FASTA or structure map with your own targets as needed.
