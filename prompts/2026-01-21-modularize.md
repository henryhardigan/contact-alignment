## Phase 3: Documentation

### 3.1 Docstring Convention (Google Style)

All functions should follow Google-style docstrings:

```python
def score_aligned(
    aln1: str,
    aln2: str,
    scorer: Callable[[str, str], float | None],
    *,
    gap_char: str = "-",
) -> tuple[float, list[float | None]]:
    """Score an aligned sequence pair position by position.

    Computes the total alignment score and per-position scores using
    the provided scoring function. Gap positions receive None scores.

    Args:
        aln1: First aligned sequence (may contain gap characters).
        aln2: Second aligned sequence (must be same length as aln1).
        scorer: Function that takes two amino acids and returns a score,
            or None for unknown/invalid pairs.
        gap_char: Character representing gaps in the alignment.

    Returns:
        A tuple containing:
            - total: Sum of all non-None position scores.
            - per_pos: List of scores for each position (None for gaps).

    Raises:
        ValueError: If aln1 and aln2 have different lengths.

    Example:
        >>> scorer = lambda a, b: -1.0 if a == b else 0.0
        >>> score_aligned("ACGT", "ACTT", scorer)
        (-2.0, [-1.0, -1.0, None, -1.0])
    """
```

### 3.2 Variable Comments Convention

Inline comments for non-obvious variables:

# Threshold for anchor detection (more negative = stronger interaction)
DEFAULT_ANCHOR_THRESHOLD: float = -25.0

# Maximum gap length in Stage 2 DP (prevents runaway gaps)
MAX_GAP_LEN: int = 10

def stage2_extend_fixed_core(...):
    # DP table dimensions: [position][gap_count][last_gap_length]
    dp_left = [[[None] * max_gap_len for _ in range(max_gaps + 1)]
               for _ in range(len1 + 1)]

    # Band constraint: only explore positions within ±band of diagonal
    band_lo = max(0, diag - band)  # Lower bound of search band
    band_hi = min(len2, diag + band)  # Upper bound of search band

    # Affine gap penalty: open + (length - 1) * extend

### 3.3 Module-Level Documentation

Each module should have a module docstring:

```python
"""Clustal-style sequence similarity scoring.

This module provides functions for computing Clustal-style similarity
between aligned protein sequences, including:

- Identity, strong similarity, and weak similarity classifications
- Position-by-position scoring with standard Clustal symbols (*, :, .)
- Anchor position detection based on similarity thresholds

The Clustal scoring follows the standard conventions:
- '*' (identity): Exact amino acid match
- ':' (strong): Amino acids in same strong similarity group
- '.' (weak): Amino acids in same weak similarity group
- ' ' (none): No similarity

References:
    - Clustal Omega: https://www.ebi.ac.uk/Tools/msa/clustalo/
    - Thompson et al. (1994) CLUSTAL W
"""
```
